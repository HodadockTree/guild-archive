# CHANGELOG

## v1.4 - 데이터 안전성 및 관리 도구 정리

### 목표

LocalStorage 기반으로 저장되는 길드 활동 데이터를 더 안전하게 관리할 수 있도록, JSON 백업 파일을 통한 복원 기능을 추가한다.

또한 길드원 등록, 백업, 복원, 복구 기능을 한곳에서 관리할 수 있도록 데이터 관리 도구의 구조를 정리한다.

### 구현

* 데이터 관리 도구 구조 정리

  * 길드원 관리
  * 백업 / 복원
  * 주의사항 영역으로 구분
* 새 길드원 등록 영역을 데이터 관리 도구 안으로 이동
* JSON 백업 내보내기 구조 보완

  * `appName`
  * `appVersion`
  * `schemaVersion`
  * `exportedAt`
  * `members`
  * `activityLogs`
* JSON 백업 가져오기 기능 추가
* 백업 파일 구조 검증 추가

  * JSON 파싱 가능 여부 확인
  * `members` 배열 확인
  * `activityLogs` 배열 확인
  * 기본 필드 경고 처리
* 복원 전 백업 파일 요약 표시

  * 앱 이름
  * 앱 버전
  * 백업 시각
  * 길드원 수
  * 활동 기록 수
* JSON 백업 파일 기반 덮어쓰기 복원 기능 추가

  * 길드원 목록 복원
  * 활동 기록 목록 복원
* 복원 전 2단계 확인 절차 추가
* 복원 완료 결과 메시지 표시
* 복원 실패/취소 시 기존 데이터가 변경되지 않도록 처리
* 기존 스프레드시트 가져오기 기능 제거

### 결과

* 브라우저 LocalStorage 데이터가 유실되더라도 JSON 백업 파일을 통해 복원할 수 있게 되었다.
* 데이터 관리 도구의 역할이 길드 데이터 관리 영역으로 명확해졌다.
* 잘못된 JSON 파일을 가져왔을 때 바로 복원되지 않고 오류를 표시하도록 개선되었다.
* 복원 시 병합이 아니라 전체 덮어쓰기 방식으로 동작해 예측 가능한 복원이 가능해졌다.
* 더 이상 사용하지 않는 스프레드시트 가져오기 기능을 제거해 관리 도구가 단순해졌다.

### 제외한 기능

* 대시보드 PNG 저장
* 병합 복원
* 활동 기록 검색 / 필터 고도화
* 길드원별 활동 이력 모달화
* 점령전 세부 통계 확장
* 서버 저장 또는 외부 DB 연동

### 검증

* `npx tsc --noEmit` 통과
* `npm run lint` 통과
* JSON 백업 내보내기 동작 확인
* 정상 백업 파일 가져오기 및 요약 표시 확인
* 잘못된 JSON 파일 오류 처리 확인
* 복원 취소 시 기존 데이터 유지 확인
* 복원 완료 후 새로고침 시 데이터 유지 확인


## v1.3.1 - 월별 정산 대시보드 및 점령전 기록 보완

### 목표

v1.3에서 구현한 월별 활동 정산 대시보드를 실제 공유해본 뒤, 캡처 화면의 가독성과 활동 기록의 의미를 보완한다.

점령전도 비공정처럼 세부 정보를 기록할 수 있게 하여, 월별 활동 기록이 더 구체적으로 남도록 개선한다.

  - 전체 활동 / 참여 길드원 / 총 참여 횟수 순서로 조정

  - 아우로라 비공정 → 아우로라
- 점령전 세부 카테고리 9종 추가

  - 평화
  - 신성
  - 지혜
  - 예언
  - 초심
  - 긍지
  - 역전

- 활동 기록 카드에 점령전 세부 카테고리 표시

  - 예: 점령전 (지혜, 초심)
- 월별 정산 대시보드의 이번 달 활동 기록에도 점령전 세부 카테고리 표시

- 길드원별 점령전 세부 통계

- 점령전 카테고리 추가 / 수정 / 삭제 관리

## v1.2

### 목표

활동 기록 시스템을 정리하고, 기존 데이터 이관과 정리 과정을 더 안정적으로 만듭니다.

과거 기록을 입력하거나 복원할 때 실수를 줄이고, 길드원별 활동 현황을 화면 안에서 빠르게 확인할 수 있게 하는 것이 목표입니다.

### 구현 기능

- 스프레드시트 가져오기에서 탈퇴일 반영
- 탈퇴일이 있는 길드원을 탈퇴 상태로 처리
- 기존 닉네임과 중복될 경우 새로 만들지 않고 기존 길드원 업데이트
- 가져오기 결과 요약 개선: 추가, 업데이트, 탈퇴 처리, 실패
- 가져오기 실패 상세 표시
- 길드원 메모 일괄 삭제
  - 활동중, 탈퇴, 전체 범위 선택
  - 실행 전 확인
  - 메모 외 데이터 보존
- 신규 활동 종류를 점령전, 비공정, 기타 중심으로 정리
- 기존 길퀘, 이벤트 기록은 데이터 호환을 위해 유지하되 표시/통계에서는 기타로 취급
- 비공정 세부 종류 추가: 오션헤븐, 아우로라
- 길드원별 활동 통계 계산 로직 분리
- 길드원 카드에 활동 요약 통계 표시: 총 참여, 점령전, 비공정, 기타
- 카드 펼침 영역에 비공정 상세 통계 표시: 오션헤븐, 아우로라
- 활동 추가/수정 폼에서 선택한 활동 종류 기준 참여 횟수 표시
- 길드원 카드의 활동 이력 보기 버튼을 카드 내부 펼침 방식으로 변경
- 카드 펼침 영역에서 활동 요약과 최근 활동 5개 표시
- 활동 추가/수정 폼에 참여자 검색 추가
- 활동중/탈퇴 길드원 영역 분리
- 활동중 길드원 기본 펼침, 탈퇴 길드원 기본 접힘
- 검색 시 활동중/탈퇴 길드원 모두 검색
- 탈퇴 길드원이 포함된 기존 활동 수정 시 선택 상태 유지
- 스프레드시트 가져오기, 가져오기 되돌리기, 메모 일괄 삭제를 데이터 관리 도구 영역으로 정리
- 데이터 관리 도구 영역 기본 접힘 처리

### 결과

v1.2에서는 스프레드시트 자료를 가져올 때 기존 길드원과 더 안전하게 병합할 수 있습니다.

탈퇴일이 있는 길드원은 자동으로 탈퇴 상태가 되고, 중복 닉네임은 새 길드원을 만들지 않고 기존 길드원 정보 업데이트로 처리됩니다.

활동 기록은 점령전, 비공정, 기타 중심으로 단순화되었고, 비공정은 오션헤븐과 아우로라를 구분해서 기록할 수 있습니다.

길드원 카드에서 총 참여, 점령전, 비공정, 기타 참여 횟수와 최근 활동을 바로 확인할 수 있습니다.

아래 항목은 아직 구현 완료가 아니라 다음 버전에서 검토할 후보입니다.

## v1.1

### 목표

과거 기록 복원과 실사용 입력 편의성을 개선합니다.

기존에 흩어져 있던 길드원 기록을 더 쉽게 가져오고, 활동 기록 작성 중 반복 입력을 줄이며, 스크린샷 기반 기록도 함께 남길 수 있게 하는 것이 목표입니다.

### 구현 기능

- 길드원 정보 수정
- 활동 기록 스크린샷 첨부
- 디스코드 이미지 Ctrl+V 첨부
- 스프레드시트 길드원 일괄 가져오기
- 일괄 가져오기 되돌리기
- 탈퇴 길드원 일괄 복구
- 참여자 목록 빈도순 정렬
- 활동중/탈퇴 길드원 분리
- 길드원 관리 영역 접기/펼치기
- 활동 제목 빠른 입력 버튼

### 결과

v1.1에서는 과거 길드원 목록을 빠르게 복원하고, 잘못 가져온 최근 일괄 추가분을 되돌릴 수 있습니다.

탈퇴 상태로 남아 있던 길드원을 한 번에 활동중으로 복구할 수 있어, 과거 기록을 다시 이어서 쓰기 쉬워졌습니다.

활동 기록에는 스크린샷을 첨부할 수 있고, 디스코드에서 복사한 이미지를 Ctrl+V로 바로 붙여넣을 수 있습니다.

참여자 선택 목록은 기존 참여 빈도가 높은 길드원이 먼저 보이도록 정렬되어 반복 입력이 줄었습니다.

- 첨부 이미지 용량 관리 방식 개선

## v1.0

### 목표

냥춘 길드 활동을 개인적으로 기록하고 다시 확인할 수 있는 LocalStorage 기반 아카이브를 만듭니다.

길드원 관리 시스템보다는 활동 기록 보존에 집중합니다.

### 구현 기능

- 길드원 등록
- 길드원 탈퇴 처리
- 탈퇴 길드원 기록 보존
- 활동 기록 추가
- 활동 종류 선택
  - 비공정
  - 점령전
  - 길드퀘
  - 이벤트
  - 기타 메모
- 활동 참여자 체크
- 전체 활동 기록 목록 보기
- 활동 종류별 필터
- 활동 기록 수정
- 활동 기록 삭제
- 길드원별 활동 이력 보기
- LocalStorage 저장

### 결과

v1.0에서는 길드 활동을 날짜, 종류, 제목, 메모, 참여자 기준으로 남기고 조회할 수 있습니다.

탈퇴한 길드원도 삭제하지 않고 상태만 변경하므로, 과거 활동 기록의 참여자 이력이 유지됩니다.
