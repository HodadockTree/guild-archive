# PROJECT_CONTEXT

## 프로젝트 개요

냥춘 길드 활동 아카이브는 테일즈런너 길드 `냥춘`의 활동 기록을 관리하고 보관하기 위한 모바일 중심 웹앱이다.

이 프로젝트는 단순 길드원 관리 시스템이 아니라, 길드 활동을 기록하고 월별로 정산해 나중에 다시 볼 수 있도록 하는 활동 아카이브를 목표로 한다.

핵심 기준은 다음과 같다.

* 실제 길드 운영에 도움이 되는가
* 활동 기록을 쉽게 입력할 수 있는가
* 기록된 데이터를 나중에 보기 쉽게 활용할 수 있는가
* 월별 정산과 공유에 쓸 수 있는 의미 있는 데이터인가
* 과도한 기능보다 자주 쓰는 기능을 우선하는가
* LocalStorage 기반 데이터 유실 위험을 줄일 수 있는가

## 현재 버전

현재 버전은 v1.4이다.

v1.3에서 월별 활동 정산 대시보드와 JSON 백업 내보내기를 추가했고, v1.3.1에서 대시보드 표시와 점령전 세부 기록을 보완했다.

v1.4에서는 JSON 백업 파일 가져오기와 덮어쓰기 복원 기능을 추가했고, 길드원 등록과 백업 / 복원 기능을 데이터 관리 도구 안으로 정리했다. 또한 더 이상 사용하지 않는 스프레드시트 가져오기 기능을 제거했다.

## 현재 주요 기능

### 길드원 관리

* 활동중 길드원과 탈퇴 길드원을 분리해서 관리한다.
* 새 길드원 등록은 데이터 관리 도구 안에서 진행한다.
* 탈퇴 길드원도 과거 활동 참여자로 기록할 수 있다.
* 활동 기록이 없는 길드원은 삭제할 수 있다.
* 탈퇴 길드원을 활동중 상태로 복구할 수 있다.
* 길드원 카드에서 활동 요약 통계를 확인할 수 있다.
* 길드원별 활동 이력을 확인할 수 있다.

### 활동 기록

* 활동 종류는 점령전 / 비공정 / 기타를 중심으로 한다.
* 기존 길퀘 / 이벤트 기록은 호환용으로 유지하되, 표시와 통계에서는 기타로 취급한다.
* 비공정은 오션헤븐 / 아우로라 세부 종류를 기록할 수 있다.
* 점령전은 9개 세부 카테고리를 기록할 수 있다.

  * 용기
  * 신념
  * 평화
  * 신성
  * 지혜
  * 예언
  * 초심
  * 긍지
  * 역전
* 점령전 세부 카테고리는 다중 선택을 지원한다.
* 점령전 세부 카테고리가 있는 경우 `점령전 (지혜, 초심)` 형태로 표시한다.
* 기존 점령전 기록은 세부 카테고리가 없어도 기존처럼 `점령전`으로 표시한다.
* 활동 추가 / 수정 폼에서 참여자를 검색할 수 있다.
* 참여자 초성 검색을 지원한다.
* 활동중 / 탈퇴 길드원을 분리해서 참여자로 선택할 수 있다.
* 참여자 목록에서 현재 활동 종류 기준 참여 횟수를 확인할 수 있다.
* 활동 기록 카드에서 참여 인원 수를 확인할 수 있다.
* 전체 활동 기록은 최신순 / 오래된순으로 정렬할 수 있다.

### 월별 활동 정산 대시보드

* 월을 선택해 해당 월의 활동을 정산할 수 있다.
* 월별 전체 활동 수를 표시한다.
* 참여 길드원 수를 표시한다.
* 총 참여 횟수를 표시한다.
* 비공정 / 점령전 / 기타 활동 통계를 표시한다.
* 기타는 0회일 때 숨김 처리한다.
* 오션헤븐 / 아우로라 비공정 세부 통계를 표시한다.
* 월간 참여 TOP N을 표시한다.
* 이번 달 활동 기록을 날짜 / 활동 종류 / 참여 인원 수 중심으로 요약한다.
* 최다 참여 활동에는 배지를 표시한다.
* 캡처해서 길드 톡방에 공유하기 좋은 레이아웃을 지향한다.

### 데이터 관리

* 데이터는 현재 LocalStorage 기반으로 저장한다.
* 데이터 관리 도구는 길드 데이터를 관리하는 영역으로 사용한다.
* 데이터 관리 도구 안에서 새 길드원을 등록할 수 있다.
* 데이터 관리 도구 안에서 탈퇴 길드원을 복구할 수 있다.
* 전체 데이터를 JSON 파일로 백업 내보내기 할 수 있다.
* JSON 백업 파일을 가져와 복원할 수 있다.
* 백업 파일을 가져올 때 구조를 검증하고, 복원 전 요약 정보를 표시한다.
* 복원 시 현재 길드원 목록과 활동 기록 목록은 백업 파일 내용으로 덮어써진다.
* 복원 전 2단계 확인 절차를 거친다.
* 복원 완료 후 복원된 길드원 수와 활동 기록 수를 메시지로 표시한다.
* 잘못된 JSON 파일이거나 필수 데이터가 없는 경우 복원하지 않고 오류를 표시한다.

## 데이터 관리 도구 구조

v1.4 기준 데이터 관리 도구는 다음 역할을 가진다.

### 길드원 관리

* 새 길드원 등록
* 탈퇴 길드원 복구
* 활동 기록 없는 길드원 삭제 안내

### 백업 / 복원

* 전체 데이터 JSON 백업 내보내기
* JSON 백업 파일 가져오기
* 백업 파일 구조 검증
* 복원 전 백업 파일 요약 표시
* 덮어쓰기 복원

### 주의사항

* 이 앱은 LocalStorage 기반으로 데이터를 저장한다.
* 브라우저 데이터 삭제, 캐시 정리, 기기 변경 등의 상황에서 데이터가 사라질 수 있다.
* 정기적으로 JSON 백업 파일을 저장해두는 것을 권장한다.
* 복원 시 현재 길드원 목록과 활동 기록 목록은 백업 파일 내용으로 덮어써진다.

## JSON 백업 / 복원

### 백업 파일 구조

v1.4 기준 백업 JSON에는 다음 정보가 포함된다.

```ts
{
  appName: "nyangchun-guild-archive",
  appVersion: "v1.4",
  schemaVersion: 1,
  exportedAt: "ISO string",
  members: [...],
  activityLogs: [...]
}
```

### 복원 대상

복원 대상은 다음 두 가지이다.

* 길드원 목록
* 활동 기록 목록

### 복원하지 않는 정보

다음 UI 상태는 복원하지 않는다.

* 현재 선택 월
* 검색어
* 정렬 상태
* 접기 / 펼치기 상태
* 기타 화면 UI 상태

### 복원 방식

v1.4의 복원 방식은 전체 덮어쓰기이다.

병합 복원은 구현하지 않는다.

병합 복원은 다음 문제가 생길 수 있기 때문에 v1.4 범위에서 제외했다.

* 중복 ID 처리
* 삭제된 길드원 복구 여부
* 수정된 활동 기록 충돌
* 탈퇴 상태 충돌
* 참여자 ID 불일치

따라서 v1.4에서는 정상 백업 파일을 선택하면 백업 요약을 먼저 보여주고, 복원 전 확인 절차를 거친 뒤 현재 길드원 목록과 활동 기록 목록을 백업 파일 내용으로 교체한다.

### 백업 파일 검증

JSON 백업 파일을 가져올 때 다음 항목을 확인한다.

필수 검증:

* JSON 파싱 가능 여부
* 최상위 값이 객체인지
* `members`가 배열인지
* `activityLogs`가 배열인지

권장 검증:

* 앱 식별 정보
* schemaVersion
* 길드원 기본 필드
* 활동 기록 기본 필드

권장 검증에서 문제가 발견된 경우에는 복원 전 경고를 표시한다.

## 데이터 구조 관련 메모

### 활동 기록

활동 기록은 기본적으로 다음 정보를 포함한다.

* id
* type
* date
* title
* participantIds
* memo
* imageDataUrl
* airshipType
* conquestTypes

`airshipType`은 비공정 세부 종류를 나타내며, 오션헤븐 / 아우로라를 구분한다.

`conquestTypes`는 점령전 세부 카테고리를 나타내며 optional 필드이다. 기존 점령전 기록에는 이 값이 없을 수 있다.

기존 데이터 호환을 위해 `conquestTypes`가 없거나 비어 있는 점령전은 일반 `점령전`으로 표시한다.

## v1.4에서 제거한 기능

### 스프레드시트 가져오기

기존 스프레드시트 가져오기 기능은 v1.4에서 제거했다.

제거 이유는 다음과 같다.

* 초기 데이터 이관용 성격이 강하다.
* 현재는 앱 내부에서 길드원과 활동 기록을 직접 관리한다.
* JSON 백업 / 복원 기능과 역할이 겹친다.
* 잘못 가져올 경우 데이터가 꼬일 위험이 있다.
* 데이터 관리 도구를 복잡하게 만든다.

앞으로 데이터 이전과 복구는 JSON 백업 / 복원 기능을 기준으로 한다.

## 현재 제외된 기능

아래 기능은 아직 구현하지 않았다.

* PNG 이미지 저장
* 공유용 텍스트 자동 생성
* CSV 내보내기
* 고급 검색 / 필터
* 점령전 세부 통계 카드
* 길드원별 점령전 세부 통계
* 점령전 세부 필터
* 점령전 카테고리 관리
* 길드원별 활동 이력 모달화
* 병합 복원
* 서버 저장
* 외부 DB 연동
