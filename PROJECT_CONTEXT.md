# PROJECT_CONTEXT

## 현재 버전

v1.3

## 프로젝트 목적

이 프로젝트는 테일즈런너 길드 "냥춘"의 활동 기록을 보관하기 위한 개인용 웹앱입니다.

길드원 관리 시스템을 만드는 것이 아니라, 길드 활동 아카이브를 만드는 것이 핵심입니다. 주요 목적은 활동 기록 입력, 길드원별 활동 통계 확인, 월별 활동 정산, 과거 기록 보관입니다.

v1.2는 활동 기록 구조와 통계 개선에 집중했고, v1.2.1은 실제 데이터 입력 중 발견한 UX 문제를 보정했습니다. v1.3은 월별 정산/공유용 대시보드와 JSON 백업 내보내기를 추가한 버전입니다.

## 중요한 개발 원칙

- 개인 실사용 흐름을 우선합니다.
- 서버 저장, 로그인, 권한 관리는 현재 구현되어 있지 않습니다.
- 데이터는 브라우저 LocalStorage에 저장합니다.
- 전체 데이터 JSON 백업 내보내기를 지원합니다.
- JSON 복원은 아직 구현되어 있지 않습니다.
- 탈퇴 길드원은 삭제하지 않고 `left` 상태로 보존합니다.
- 삭제 기능은 오타나 중복 등록처럼 잘못 만든 길드원 데이터를 정리하는 용도입니다.
- 과거 활동 기록의 참여자 이력이 깨지지 않도록 기존 `id`를 유지합니다.
- 구현되지 않은 기능을 문서나 UI에서 구현된 것처럼 표현하지 않습니다.
- 기능은 필요한 만큼만 작게 추가합니다.
- Next.js 관련 코드를 수정할 때는 `node_modules/next/dist/docs/`의 현재 버전 문서를 먼저 확인합니다.

## 월별 정산 대시보드

월별 정산 대시보드는 활동 기록을 월 단위로 모아 캡처 공유하기 좋은 형태로 보여줍니다.

현재 표시하는 항목은 다음과 같습니다.

- 월별 전체 활동 수
- 총 참여 횟수
- 참여 길드원 수
- 점령전 / 비공정 / 기타 통계
- 오션헤븐 / 아우로라 비공정 세부 통계
- 월간 참여 TOP 5
- 이번 달 활동 기록 요약

공유용 텍스트 생성, 이미지/PNG 저장은 아직 구현되어 있지 않습니다. 현재는 화면을 직접 캡처해 공유하는 흐름을 전제로 합니다.

## 활동 통계

길드원별로 다음 활동 요약 통계를 계산합니다.

- 총 참여
- 점령전
- 비공정
- 기타
- 오션헤븐
- 아우로라

길드원 카드 내부 펼침 영역은 상세 통계 중심으로 단순화되어 있습니다. 하단의 길드원별 활동 이력 영역은 아직 유지됩니다.

## 핵심 데이터 구조

### GuildMember

위치: `src/types.ts`

```ts
export type GuildMemberStatus = "active" | "left";

export interface GuildMember {
  id: string;
  nickname: string;
  status: GuildMemberStatus;
  joinedAt: string;
  leftAt: string | null;
  memo?: string;
}
```

- `status: "active"`는 현재 활동 중인 길드원입니다.
- `status: "left"`는 탈퇴 처리된 길드원입니다.
- 탈퇴 처리 시 `leftAt`에 날짜가 저장됩니다.
- 길드원 정보 수정이나 스프레드시트 가져오기 업데이트 시 기존 `id`를 유지해야 과거 활동 기록 연결이 유지됩니다.
- 메모 일괄 삭제는 `memo`만 비우고 `id`, `status`, `joinedAt`, `leftAt`은 보존해야 합니다.
- 활동 기록이 있는 길드원은 삭제하지 않습니다.
- 길드원 삭제는 탈퇴 처리와 다릅니다. 삭제는 오타/중복 등록처럼 잘못 만든 데이터 정리용입니다.

### ActivityLog

위치: `src/types.ts`

```ts
export type ActivityType =
  | "airship"
  | "siege"
  | "guildQuest"
  | "event"
  | "other";

export type AirshipType = "ocean" | "aurora";

export interface ActivityLog {
  id: string;
  type: ActivityType;
  airshipType?: AirshipType;
  date: string;
  title?: string;
  participantIds: string[];
  memo?: string;
  imageDataUrl?: string;
}
```

- `participantIds`에 참여 길드원의 `id`를 저장합니다.
- 길드원 닉네임은 활동 기록에 복사 저장하지 않고, 길드원 목록에서 `id`로 찾아 표시합니다.
- `type`은 기존 데이터 호환 때문에 `guildQuest`, `event`를 유지할 수 있습니다.
- 신규 입력 UX와 주요 화면 표현은 `siege`, `airship`, `other` 중심입니다.
- 기존 `guildQuest`, `event` 기록은 표시와 통계에서 `other`로 취급합니다.
- `airshipType`은 비공정 세부 종류입니다. `ocean`은 오션헤븐, `aurora`는 아우로라입니다.
- `imageDataUrl`은 활동 기록에 첨부된 이미지를 저장합니다.
- 이미지는 LocalStorage에 함께 저장되므로 용량이 커질 수 있습니다.

### LocalStorage 키

- `guild-archive:members`
- `guild-archive:activities`

참여 기록은 별도 테이블로 저장하지 않고, 각 `ActivityLog`의 `participantIds` 배열에 포함됩니다.

## v1.3 작업 맥락

v1.3은 활동 입력 자체보다 월별 정산과 데이터 백업에 초점을 둔 버전입니다.

`src/lib/monthlyReport.ts`는 선택한 월의 활동 기록을 기준으로 전체 활동 수, 총 참여 횟수, 참여 길드원 수, 활동 종류별 통계, 비공정 세부 통계, 월간 참여 TOP 5, 활동 기록 요약을 계산합니다.

월별 정산 영역은 길드 톡방 공유를 위해 캡처하기 쉬운 레이아웃을 목표로 합니다. 다만 앱 안에서 공유용 텍스트를 만들거나 PNG 파일을 저장하는 기능은 아직 없습니다.

JSON 백업 내보내기는 현재 LocalStorage에 저장된 길드원과 활동 기록을 파일로 저장하는 기능입니다. JSON 복원은 아직 구현되어 있지 않으므로 백업 파일을 다시 앱으로 가져오는 흐름은 다음 버전 후보입니다.

- 활동 기록 검색/필터 고도화

## 주의할 점

- 현재 데이터는 브라우저 LocalStorage에만 있습니다.
- 서버 저장, 로그인, 권한 관리는 아직 없습니다.
- JSON 백업 내보내기는 있지만 JSON 복원은 아직 없습니다.
- 브라우저 저장소 초기화, 다른 브라우저 사용, 시크릿 모드에서는 기록이 유지되지 않을 수 있습니다.
- 이미지도 LocalStorage에 저장되므로 많은 스크린샷을 첨부하면 저장 용량 제한에 걸릴 수 있습니다.
- 탈퇴 길드원을 삭제하면 과거 활동 이력이 깨질 수 있습니다.
- 활동 기록이 있는 길드원은 삭제하지 않아야 합니다.
- 스프레드시트 가져오기에서 닉네임이 중복되면 새로 만들지 말고 기존 길드원을 업데이트하는 흐름을 유지해야 합니다.
- 길드원 메모 일괄 삭제는 메모만 삭제하고 활동 이력, 상태, 가입일, 탈퇴일은 보존해야 합니다.
- 탈퇴 길드원이 포함된 기존 활동 수정 시 선택 상태가 유지되어야 합니다.
- 검색 중에도 이미 선택한 참여자 상태가 유지되어야 합니다.
- `guildQuest`, `event`는 기존 데이터 호환을 위해 타입에 남아 있으며 표시와 통계에서는 기타로 취급합니다.
- `src/lib/activities.ts`는 과거 필드명 `participantMemberIds`를 `participantIds`로 읽어오는 호환 처리를 포함합니다.
