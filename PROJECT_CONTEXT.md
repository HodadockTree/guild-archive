# PROJECT_CONTEXT

## 프로젝트명

냥춘 길드 활동 아카이브

## 현재 버전

v1.5

## 프로젝트 개요

냥춘 길드 활동 아카이브는 테일즈런너 길드 `냥춘`의 활동 기록을 관리하고 보관하기 위한 모바일 중심 웹앱이다.

이 앱은 단순 길드원 관리 시스템이 아니라, 길드 활동을 기록하고 월별로 정산해 나중에 다시 볼 수 있도록 하는 **길드 활동 아카이브**를 목표로 한다.

## 핵심 방향

현재 프로젝트의 핵심 방향은 다음과 같다.

1. 실제 길드 운영에 도움이 되는가
2. 활동 기록을 쉽게 입력할 수 있는가
3. 기록된 데이터를 나중에 보기 쉽게 활용할 수 있는가
4. 월별 정산과 공유에 쓸 수 있는 의미 있는 데이터인가
5. 과도한 기능보다 자주 쓰는 기능을 우선하는가
6. LocalStorage 기반 데이터 유실 위험을 줄일 수 있는가
7. 향후 길드 연혁으로 확장 가능한 기록 구조를 유지하는가

## 현재 구현 상태

### 길드원 관리

- 활동중 길드원과 탈퇴 길드원을 분리해서 관리한다.
- 탈퇴 길드원도 과거 활동 참여자로 기록할 수 있다.
- 활동 기록이 없는 길드원은 삭제할 수 있다.
- 길드원별 활동 요약 통계를 확인할 수 있다.
- 길드원별 활동 이력은 모달 방식으로 확인한다.
- 활동 이력이 많은 길드원을 선택해도 메인 화면 레이아웃이 과도하게 길어지지 않도록 개선되었다.

### 활동 종류

사용자 화면 기준 활동 종류는 다음을 중심으로 정리한다.

- 점령전
- 비공정
- 이벤트

v1.5부터 기존 `기타` 활동은 사용자 화면에서 `이벤트`로 표시한다.

다만 기존 LocalStorage 데이터와 JSON 백업 호환을 위해 내부 타입과 기존 기록 구조는 유지한다.

기존 버전에서 사용되던 `길퀘`, `이벤트`, `기타` 성격의 과거 기록은 호환성을 고려해 유지한다.

### 비공정 기록

비공정은 다음 세부 종류를 기록할 수 있다.

- 오션헤븐
- 아우로라

입력 시 버튼형 UI로 선택할 수 있다.

월별 정산과 활동 기록 카드에서는 세부 종류가 표시된다.

### 점령전 기록

점령전은 다음 세부 카테고리를 기록할 수 있다.

- 용기
- 신념
- 평화
- 신성
- 지혜
- 예언
- 초심
- 긍지
- 역전

점령전 세부 카테고리는 다중 선택할 수 있다.

활동 기록 카드와 월별 대시보드에서는 `점령전 (지혜, 초심)` 형태로 표시한다.

기존 점령전 기록은 세부 카테고리가 없어도 기존처럼 `점령전`으로 표시한다.

v1.5부터 점령전 빠른 제목 버튼은 제거하고, 세부 카테고리 선택 중심으로 입력 UX를 정리했다.

기존 `title` 데이터와 제목 직접 입력 기능은 호환성을 위해 유지한다.

### 활동 기록

활동 기록은 다음 정보를 중심으로 관리한다.

- 활동 종류
- 비공정 세부 종류
- 점령전 세부 카테고리
- 날짜
- 제목
- 참여자
- 메모
- 이미지

활동 기록은 최신순 / 오래된순으로 정렬할 수 있다.

활동 기록 추가 / 수정 / 삭제 시 완료 메시지를 표시한다.

활동 기록 카드에서는 참여 인원 수를 확인할 수 있다.

### 참여자 선택

활동 추가 / 수정 시 참여자를 선택할 수 있다.

- 활동중 길드원과 탈퇴 길드원을 분리해서 선택한다.
- 탈퇴 길드원도 과거 활동 참여자로 기록할 수 있다.
- 참여자 검색을 지원한다.
- 초성 검색을 지원한다.
- 선택한 참여자 수를 표시한다.
- 참여자 목록에서 현재 활동 종류 기준 참여 횟수를 표시한다.

### 월별 활동 정산

월별 활동 정산 대시보드에서 선택한 월의 활동 데이터를 확인할 수 있다.

표시 정보는 다음과 같다.

- 전체 활동 수
- 총 참여 횟수
- 참여 길드원 수
- 비공정 / 점령전 / 이벤트 통계
- 오션헤븐 / 아우로라 비공정 세부 통계
- 점령전 세부 카테고리 통계
- 월간 참여 TOP
- 이번 달 활동 기록 요약
- 최다 참여 활동 배지

월별 정산 카드 영역은 캡처 공유용으로 활용할 수 있도록 구성한다.

### 월간 공유 문구

v1.5부터 월별 정산 설정 영역에서 공유 문구를 복사할 수 있다.

공유 문구에는 다음 정보가 포함된다.

- 월간 활동 정산 제목
- 전체 활동 수
- 총 참여 횟수
- 참여 길드원 수
- 활동별 기록
  - 비공정
  - 점령전
  - 이벤트
- 참여 TOP
- 이번 달 이벤트 목록

공유 문구 복사 버튼은 캡처용 월별 정산 카드 안이 아니라 월별 정산 설정 영역에 배치한다.

이는 캡처용 결과물 영역에 조작 버튼이 보이지 않도록 하기 위한 것이다.

### 길드원 활동 이력 모달

v1.5부터 길드원 활동 이력은 메인 화면 아래에 펼치는 방식이 아니라 모달 방식으로 확인한다.

기존 방식은 특정 길드원의 활동 기록이 많을 경우 활동 이력 영역이 지나치게 길어져 전체 페이지 레이아웃이 밀리는 문제가 있었다.

모달 방식으로 변경하면서 다음을 개선했다.

- 메인 화면 레이아웃 유지
- 모바일 사용성 개선
- 활동 이력이 많은 길드원도 모달 내부에서 확인 가능
- 길드원 활동 요약과 활동 이력 목록을 분리해 확인 가능

모달에서는 다음 정보를 확인할 수 있다.

- 길드원 닉네임
- 전체 참여 횟수
- 비공정 참여 횟수
- 점령전 참여 횟수
- 이벤트 참여 횟수
- 활동 이력 목록

### 데이터 관리

데이터는 브라우저 LocalStorage를 기반으로 저장한다.

LocalStorage는 브라우저 데이터 삭제, 기기 변경, 시크릿 모드 사용 등에 따라 데이터가 사라질 수 있으므로 백업 기능이 중요하다.

v1.4부터 JSON 백업 / 복원 기능을 제공한다.

지원 기능은 다음과 같다.

- 전체 데이터 JSON 백업 내보내기
- JSON 백업 파일 가져오기
- 백업 파일 구조 검증
- 복원 전 백업 파일 요약 표시
- JSON 백업 파일 기반 덮어쓰기 복원
- 복원 전 2단계 확인 절차
- 복원 완료 결과 메시지 표시
- 잘못된 파일 / 복원 취소 시 기존 데이터 유지

v1.5에서는 내부 데이터 구조나 schemaVersion을 변경하지 않고, 기존 데이터 호환을 유지한다.

## v1.5 변경 요약

v1.5는 월간 정산 공유와 활동 이력 UX 개선을 중심으로 한 버전이다.

주요 변경 사항은 다음과 같다.

- 기존 `기타` 활동 표시명을 사용자 화면에서 `이벤트`로 정리
- 월별 활동 정산 공유 문구 생성 기능 추가
- 월별 정산 설정 영역에 공유 문구 복사 버튼 추가
- 공유 문구에 전체 활동 수, 총 참여 횟수, 참여 길드원 수, 활동별 기록, 참여 TOP, 이번 달 이벤트 목록 포함
- 점령전 빠른 제목 버튼 제거
- 점령전 세부 카테고리 선택 중심으로 입력 UX 정리
- 길드원 활동 이력 보기를 모달 방식으로 변경
- 활동 이력이 많은 길드원 선택 시 메인 화면 레이아웃이 길어지는 문제 개선
- 캡처용 월별 정산 카드 안에 조작 버튼이 보이지 않도록 정리

## v1.5에서 제외한 것

v1.5에서는 다음 기능을 구현하지 않는다.

- PNG 저장 기능
- 별도 길드 연혁 페이지
- 내부 ActivityType을 `event`로 완전 마이그레이션
- schemaVersion 변경
- 대시보드 상단 긴 리포트 문장 상시 표시
- 복잡한 자동 하이라이트 문장 생성
- 길드원별 월간 비교 / 랭킹 상세 분석

## 향후 고려 사항

v1.5 이후에는 다음 기능을 검토할 수 있다.

### 공유 문구 개선

현재 공유 문구는 월별 정산 데이터를 빠르게 복사하는 데 초점을 둔다.

향후 실제 길드 톡방에 공유해본 뒤, 문구의 톤과 구성을 더 자연스럽게 다듬을 수 있다.

### 이벤트 기록 모아보기

기존 `기타`를 사용자 화면에서 `이벤트`로 정리했기 때문에, 향후 이벤트 기록만 모아보는 기능을 추가할 수 있다.

예상 기능은 다음과 같다.

- 이벤트 활동만 필터링
- 월별 이벤트 목록 보기
- 이벤트 제목 / 메모 / 참여자 / 이미지 중심 표시

### 길드 연혁 페이지

이벤트 기록이 충분히 쌓이면 길드 연혁 페이지로 확장할 수 있다.

예상 기능은 다음과 같다.

- 연도별 / 월별 주요 이벤트 보기
- 대운동회, 길드 마크 공모전, 합동 달리기 같은 주요 기록 보관
- 이미지와 메모를 포함한 기록 카드
- 길드 활동 히스토리 아카이브

### 데이터 구조 정리

향후 필요하다면 schemaVersion 기반 마이그레이션을 도입할 수 있다.

그 시점에 내부 타입에서 과거 `기타` 계열을 `이벤트`로 정식 전환하는 방안을 검토한다.

다만 v1.5에서는 데이터 안정성을 우선해 내부 구조를 변경하지 않는다.
